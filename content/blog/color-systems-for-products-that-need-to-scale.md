A practical approach to building a color palette that works in light mode, dark mode, and every brand surface in between.

## Beyond the Hand-Picked Palette

Most early-stage products start with a palette of six swatches picked by eye. This works until the product grows past a single screen. After that, the palette starts to crack and inconsistencies multiply.

## Think in Roles, Not Hex Codes

A scalable color system names colors by what they do, not what they look like. Surface, foreground, accent, success, warning, and destructive are roles. The hex values can change without breaking the product.

- Surface colors for backgrounds
- Foreground colors for text
- Accent colors for emphasis
- Semantic colors for status

## Plan for Dark Mode From the Start

Retrofitting dark mode after launch is painful and expensive. Designing in both modes from the first frame is dramatically cheaper and produces a better result in both directions.

## Use Tokens to Bridge Design and Code

Design tokens let designers and engineers speak the same language. When the design file says 'surface-2' and the code says the same thing, conversations get shorter and bugs disappear.

## Accessibility Is Built In, Not Bolted On

Check contrast as the system is built, not after. Every paired combination of foreground and background should pass the relevant contrast threshold by default.
